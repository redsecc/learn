A condensed enumeration pass for Linux privilege escalation. Run top to bottom;
anything unusual is a lead. Pair this with the
[Linux privilege escalation](/paths/linux-privesc) path for the *why* behind each
check.

## Who and where am I

```bash
id
whoami
hostname
uname -a            # kernel version → known exploits?
cat /etc/os-release
```

## SUID / SGID binaries

```bash
find / -perm -4000 -type f 2>/dev/null   # SUID
find / -perm -2000 -type f 2>/dev/null   # SGID
```

Cross-reference unexpected results with GTFOBins.

## Sudo rights

```bash
sudo -l                  # what can I run as another user?
sudo -V | head -1        # sudo version → known CVEs?
```

## Capabilities

```bash
getcap -r / 2>/dev/null  # cap_setuid / cap_dac_read_search are gold
```

## Writable & interesting files

```bash
# world-writable files and dirs
find / -writable -type f 2>/dev/null | grep -vE '^/proc|^/sys'
# can I read sensitive files?
ls -la /etc/shadow /etc/passwd
# config files with secrets
grep -RInE 'password|secret|api[_-]?key' /var/www /opt /home 2>/dev/null
```

## Cron jobs

```bash
cat /etc/crontab
ls -la /etc/cron.* /var/spool/cron 2>/dev/null
```

A cron job running a script you can write to is a classic root path.

## Processes, services, networking

```bash
ps aux --forest
ss -tulpn          # listening services (internal ones are interesting)
```

## Automate the grind

```bash
# transfer and run, then read the output critically
./linpeas.sh | tee linpeas.out
```
