A fast lookup for the ports that show up in almost every engagement. Use it to
turn an `nmap` result into a plan.

## Scan first

```bash title="Full TCP port discovery, then service/version detection"
nmap -p- --min-rate 2000 -T4 <target> -oA tcp-all
nmap -sCV -p <open-ports> <target> -oA tcp-services
```

## Frequently seen ports

| Port | Service | First thing to check |
| ---- | ------- | -------------------- |
| 21 | FTP | Anonymous login (`anonymous:anonymous`), writable dirs |
| 22 | SSH | Version/CVEs, weak creds, key-based auth |
| 23 | Telnet | Cleartext creds, default logins |
| 25 | SMTP | Open relay, `VRFY`/`EXPN` user enumeration |
| 53 | DNS | Zone transfer (`AXFR`), subdomain leaks |
| 80 | HTTP | Full web recon — content, params, stack |
| 110 | POP3 | Cleartext creds |
| 111 | RPCbind | `rpcinfo`, NFS exports |
| 135 | MSRPC | Endpoint mapper enumeration |
| 139 / 445 | SMB | Null sessions, shares, signing, EternalBlue-era CVEs |
| 143 | IMAP | Cleartext creds |
| 161 | SNMP | Default community strings (`public`) |
| 389 / 636 | LDAP(S) | Anonymous bind, directory enumeration |
| 443 | HTTPS | Web recon + TLS config/cert data |
| 1433 | MSSQL | Weak `sa`, `xp_cmdshell` |
| 2049 | NFS | `showmount -e`, no_root_squash |
| 3306 | MySQL | Weak creds, reachable from outside |
| 3389 | RDP | Weak creds, NLA, BlueKeep-era CVEs |
| 5432 | PostgreSQL | Weak creds, `COPY ... PROGRAM` |
| 5985 / 5986 | WinRM | Valid creds → remote shell (`evil-winrm`) |
| 6379 | Redis | Unauthenticated access, write to disk |
| 8080 | HTTP-alt | Proxies, dev servers, admin panels |
| 27017 | MongoDB | Unauthenticated access |

## Quick checks by service

```bash title="SMB"
smbclient -L //<target>/ -N
nxc smb <target> --shares
```

```bash title="DNS zone transfer"
dig axfr @<target> example.com
```

```bash title="SNMP"
snmpwalk -v2c -c public <target>
```
