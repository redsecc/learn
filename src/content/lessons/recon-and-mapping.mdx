Every web engagement starts with reconnaissance. You can't exploit what you
haven't found, and most real findings come from surface you mapped that the
defenders forgot about — old endpoints, staging hosts, hidden parameters.

## Enumerate subdomains

Start broad. Passive sources give you hostnames without touching the target:

```bash title="Passive subdomain enumeration"
subfinder -d example.com -silent | tee subs.txt
```

Then resolve which ones are actually live:

```bash title="Probe for live hosts"
cat subs.txt | httpx -silent -status-code -title
```

## Map content and endpoints

Crawl the application and brute-force for hidden paths:

```bash title="Directory and content discovery"
ffuf -u https://example.com/FUZZ \
     -w /usr/share/seclists/Discovery/Web-Content/raw.txt \
     -mc 200,204,301,302,401,403 \
     -o ffuf.json
```

<Callout type="tip" title="Read before you fuzz">
  View source, walk the JavaScript bundles, and read the
  <code>robots.txt</code>, <code>sitemap.xml</code>, and any API schema. Apps
  routinely leak endpoint names, parameter names, and internal hostnames in
  client-side code.
</Callout>

## Identify the stack

Fingerprint the technology so you know which bug classes to prioritize:

```bash title="Fingerprinting"
whatweb https://example.com
```

Framework, server, and CMS all change your plan: a Rails app, a WordPress
install, and a bespoke PHP site each fail in different, predictable ways.

## Build the map

Keep a living inventory as you go:

- **Hosts** — every live subdomain and IP.
- **Endpoints** — each route, method, and required parameters.
- **Inputs** — every place user data enters: query strings, headers, cookies,
  JSON bodies, file uploads.
- **Trust boundaries** — where authn/authz is enforced, and where it isn't.

<Callout type="warning" title="Stay in scope">
  Only test hosts that are explicitly in scope for your engagement. Recon tools
  make it easy to wander onto third-party infrastructure — confirm ownership
  before you send a single request.
</Callout>

## Recap

- Enumerate subdomains, then probe for live hosts.
- Discover content and hidden parameters with fuzzing plus source review.
- Fingerprint the stack and maintain a living map of inputs and trust
  boundaries.
